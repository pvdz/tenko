> :: functions : declaration : block scoped
>
> ::> nested block function then var in inner fail
>
> A let declaration followed by a block function with same name in inner block: the function's implicit var conflicts with the let.
>
> In sloppy mode (both with and without annexb), the block function creates an implicit var that conflicts with the outer let. Without annexb it throws; with annexb the var is skipped (B.3.3.1).

`````js
(function() {
  {
    let f;
    {
      function f() {}
    }
    f;
  }
`````
